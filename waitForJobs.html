Wait for termination of jobs on the batch system.

Usage

waitForJobs(reg, ids, sleep = 10, timeout = 604800, stop.on.error = FALSE)

Arguments

reg
[Registry] Registry.
ids
[integer] Vector of job ids. Default is all jobs currently on the system.
sleep
[numeric(1)] Seconds to sleep between status updates. Default is 10.
timeout
[numeric(1)] After waiting timeout seconds, show a message and return FALSE. This argument may be required on some systems where, e.g., expired jobs or jobs on hold are problematic to detect. If you don't want a timeout, set this to Inf. Default is 604800 (one week).
stop.on.error
[logical(1)] Immediatly return if a job terminates with an error? Default is FALSE.

Value

Returns TRUE if all jobs terminated successfully and FALSE if either an error occured or the timeout is reached.

Description

Waits for termination of jobs while displaying a progress bar containing summarizing informations of the jobs. The following abbrevations are used in the progress bar: “S” for number of jobs on system, “R” for number of jobs running “D” for number of jobs successfully terminated and “E” for number of jobs terminated with an R exception.
